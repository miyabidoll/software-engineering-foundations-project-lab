### Performance (Desempenho)

-	O sistema deve processar a consulta de um produto no banco de dados em até 1,5 segundo após o escaneamento.
-	O terminal deve ser capaz de realizar sincronização de estoque com o servidor central em intervalos de 5 minutos.

---

### Security (Segurança)

-	O sistema deve exigir autenticação segura para que apenas usuários registrados realizem compras.
-	Os dados de transações financeiras devem ser criptografados e não podem ficar armazenados localmente no totem.
-	O sistema deve possuir níveis de permissão diferentes para "aluno" (comprador) e "repositor" (acesso ao estoque).

---

### Usability (Usabilidade)

- A interface do totem deve ser adaptada para telas touch screen com botões grandes e legíveis.
- O usuário deve conseguir finalizar uma compra completa de até 3 itens em menos de 45 segundos.

---

## 4. User Stories (Histórias de Usuário)

-	Como uma estudante do turno da noite, eu quero comprar um item de higiene pessoal dentro da faculdade, para que eu não precise sair na rua e me expor a riscos de segurança.
-	Como um aluno com pressa entre uma aula e outra, eu quero pagar minha compra rapidamente via PIX, para que eu não chegue atrasado na próxima aula.
-	Como um responsável pela reposição, eu quero visualizar quais produtos estão com estoque baixo no sistema, para que eu possa reabastecer a prateleira de forma eficiente.
