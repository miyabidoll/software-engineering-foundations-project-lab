## 5. Assumptions (Premissas)

-	Assume-se que o local de instalação no campus possui ponto de energia e rede Wi-Fi estável para o terminal.
-	Assume-se que os alunos possuem conta em banco ou carteira digital para realizar pagamentos via PIX ou cartão.
-	Assume-se que a faculdade autorizará o uso do espaço físico e a venda de produtos de higiene e lanches.
-	Assume-se que os usuários agirão com honestidade no modelo de self-checkout, respeitando a conferência dos itens.

---

## 6. Constraints (Restrições)

-	O sistema deve rodar de forma leve, pois o hardware do totem de autoatendimento terá recursos limitados de processamento.
-	O projeto não pode utilizar métodos de pagamento que cobrem mensalidades fixas, devendo priorizar taxas por transação para viabilidade financeira.
-	O sistema deve seguir as normas de segurança e privacidade de dados (LGPD) ao manipular informações de cadastro dos alunos.
-	O estoque físico é limitado ao tamanho do armário/dispensa disponível no campus, restringindo a variedade de produtos.
-	O software deve ser desenvolvido para ser operado via tela touch, dispensando o uso obrigatório de teclado físico no terminal.
