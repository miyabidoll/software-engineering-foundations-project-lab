## 5. Assumptions (Premissas)

- Assume-se que os usuários terão acesso à internet.
- Assume-se que o sistema será utilizado via dispositivos mobile e navegador web.
- Assume-se que os alunos fornecerão informações corretas no cadastro (idade e ano escolar) para personalização adequada do ensino.
- Assume-se que os usuários podem fazer parte de qualquer faixa etária.
- Assume-se que as trilhas seguem os padrões de conteúdo do ensino fundamental do MEC e BNCC.
- Assume-se que os professores irão utilizar o sistema para criação e gerenciamento de grupos/turmas.
- Assume-se que os dispositivos utilizados (celular, tablet ou computador) possuem capacidade mínima para executar o sistema de forma adequada.
- Assume-se que haverá infraestrutura tecnológica suficiente para suportar múltiplos usuários simultaneamente.

---

## 6. Constraints (Restrições)

- O sistema deve operar com tempo de resposta máximo de até 3 segundos em condições normais.
- O sistema deve suportar acesso simultâneo de grande quantidade de usuários (ex: 10.000+), limitado pela infraestrutura disponível.
- O sistema deve garantir a proteção de dados dos usuários, atendendo às normas de segurança e privacidade.
- O sistema deve implementar autenticação segura, incluindo criptografia de senhas.
- O acesso às funcionalidades deve ser controlado por níveis de permissão (aluno, professor e administrador).
- O sistema deve ser compatível com diferentes dispositivos (celular, tablet e computador), respeitando limitações tecnológicas de cada plataforma.
- O sistema depende de conexão com a internet para funcionamento.
- O conteúdo e funcionalidades do sistema devem ser adequados ao público-alvo (alunos do 6º ao 9º ano).
- O sistema deve seguir padrões de usabilidade, permitindo que ações principais sejam realizadas com até 3 interações.
- As atualizações de progresso, pontuação e ranking devem ocorrer em tempo quase real, limitadas pela capacidade do sistema.
